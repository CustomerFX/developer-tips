FSCopy.exe

FSCopy.exe is a command-line utility that allows you to import or export between Saleslogix supported file system types (VFS, ZIP, disk). For information on the arguments of this utility, type the utility name at a command line prompt in the Saleslogix install directory on the Administrative Workstation.

Before running FSCopy.exe to copy translated resources into the database, ensure that Application Architect is closed; otherwise, the build Web platform process does not recognize the changes or add them to the Web site. After the FSCopy.exe utility runs, open Application Architect. Build and deploy. See the Application Architect online help topics "Building the Web Platform" and "Deploying the Saleslogix Web Client Portal".

 

The following table describes the most common usage scenarios.

 

Usage Scenario

 Command-line Options

Get help

fscopy/?

Export entire VFS

fscopy/source:VFS:\\ /target:c:\

Export a directory from VFS

fscopy/source:VFS:\Model /target:c:\

Zip the VFS directory

fscopy/source:VFS:\Model /target:C:\

Export .resx files from the VFS model for localization

fscopy /source:"VFS:\Model" /target:"C:\ModelLocalization /recursive:true /filter:*.resx /connection:MyConnectionString

Import .resx files into the VFS model

fscopy /source:"C:\ModelLocalization" /target:"VFS:\Model" /recursive:true /filter:"*.resx" /connection:"MyConnectionString"

Get help on a custom FSCopy definition file (xml file that defines a set of copy operations)

fscopy "MyCustomExport.xml" /?

Rebuild VFS locally (simulating VFS build process)

fscopy "..\ci\FSCOPY_Build.xml /target:VFS: /root:.. /bins:..\debug /batch:false /truncate:true /connection:MyConnectionString
