Building Custom Workflows

Use Visual Studio Workflow Designer to create workflows. Use any mix of Windows Workflow activities, Saleslogix custom activities, and activities you create. You can bind your workflow to any Saleslogix entity.

Before you begin creating workflows in Visual Studio, install Saleslogix VS workflow extensions.

To build a workflow

  1. Create your workflow project.
    1. Launch Visual Studio 2008.
    2. On the File menu, click New Project.
    3. Select .NET Framework 3.0 (or greater) in the drop-down list.
    4. Expand Visual C#, and select a project type of Workflow.
    5. Select Sequential Workflow Library.
    6. Name your workflow.
    7. Click OK.
  2. Add references in the project to Saleslogix assemblies.
    1. Right-click on the References node in your new Visual Studio project, and click Add Reference.
    2. On the Browse tab, navigate to your folder of consolidated assemblies, for example, C:\Program Files\SalesLogix\ReferenceAssemblies.
    3. Select the following files and click OK.
      • Sage.Platform.dll
      • Sage.Platform.Process.dll
      • Sage.Entity.Interfaces.dll
    4. Click OK.

      Ensure this copy of Sage.Entity.Interfaces.dll is updated automatically each time you build interfaces or the platform. Refer to the Installing SalesLogix VS Workflow Items help topic for more information on setting  the Assemblies Path build setting to point to the consolidation folder.

  3. Change the base class of the workflow.
    1. Click in the workflow, click the Base Class box in the Property grid, and click the ellipsis button.

      The Browse and Select a .NET type dialog box opens.

    2. Expand the Sage.Platform.Process node, select Sage.Platform.Process.Activities, and then select SequentialEntityWorkflow or GoalDrivenEntityWorkflow.

      The ProcessTasks tasklet uses this information to identify the goal-driven processes.

    3. Click OK.
  4. Bind the workflow to an entity.
    1. Right-click anywhere in the Workflow Designer and click Select Entity.
    2. Select the entity type to which you want your workflow bound, and click OK.

      The header of the Workflow Designer indicates that your workflow is now bound to the entity type you selected. Notice also the code-behind generated for your workflow:  

      public Sage.Entity.Interfaces.ILead CurrentEntity   

      {   

          get  

          {   

              return ((Sage.Entity.Interfaces.ILead)(this.GetEntity()));   

          }   

      }

       

      This code will be useful later when you reference the current entity from policies and other workflow activities.

  5. If necessary, create custom security entities for use by your policy activities.
  6. Add Windows Workflow activities and Saleslogix custom activities. Activities to consider using include:
    • Windows Policy Activities, good for encoding business rules.
    • Saleslogix CreateToDo and CreatePhoneCall activities, often used in sequential workflows.
    • Saleslogix GoalDrivenActivity, GoalActivity, and GoalDrivenScopeActivity, for goal-driven workflows.
  7. Save the changes, and build the Visual Studio project.
