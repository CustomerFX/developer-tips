Pre-Conditions in Process Orchestration

The database eventing mechanism notifies the Process Orchestration engine that an entity instance has been changed, inserted, or deleted. After that, the process can run right away or evaluate conditions on the entity instance to determine if the process should run. For example, database eventing could fire when the Division property of an Account changes, and then the Process Host engine would start the process only if Account.State equals 'AZ' .

Pre-condition expressions are evaluated by the Saleslogix Expression Language evaluator. Refer to the Application Architect help topic Expression Language Features for more information. String values must be in quotes, and if there are multiple conditions, each must be enclosed with parenthesis. Some examples of conditions are:

 

condition = " Division='HQ'"

condition = "(Closed = true) and (SalesPotential > 1000000)"

condition = "Employees=45"

condition = ""

To enter a pre-condition in the processStart.xml file, just add it to the end of the entity type node.  In the following example, when an Account's Division property value has changed, the Division property will be checked to verify the value is equal to 'HQ' before starting the process:

 

 

<?xml version="1.0" encoding="utf-8"?>

<processStartConfiguration xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xmlns:xsd="http://www.w3.org/2001/XMLSchema"

    <event name="PropertyChanged">

        <entityType id="63b61866-48ec-422b-b2bb-09a199b26cd8" name="Sage.Entity.Interfaces.IAccount, Sage.Entity.Interfaces" memberFilter="Division" condition="Division='HQ'">

            <runProcess name="DivisionSendMail" />

        </entityType>

    </event>

</processStartConfiguration>
