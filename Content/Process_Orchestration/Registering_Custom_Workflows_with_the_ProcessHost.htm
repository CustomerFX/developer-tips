Registering Custom Workflows with the ProcessHost

After you build custom workflows and set up the Process Host portal, you can register the workflows with it.

To register

  1. Add the workflow assembly to the project in the VFS:

  1. Expand  Portal Manager > Process Host portal > SupportFiles.

  2. Right-click on bin, and click Add Existing Files.

  3. Browse to and select the workflow library that you created in Visual Studio.

  1. Register the workflow in Application Architect by doing the following:

  1. Expand Portal Manager > Process Host Portal > SupportFiles.

  2. Double-click on ProcessLib.xml to open it in the editor.

  3. Add an <assembly name> entry according to the example in the file.

  4. Save your changes.

  5. Browse to http://servername:port/ProcessHost/ProcessManager/crm/-/Processes to start the MS Workflow run time and verify the new assembly was added.
