Enabling the Process Tasks Tasklet

The Process Tasks Tasklet permits users to mark goal-driven workflow items as complete in the Saleslogix Web Client. The tasklet is available in the Saleslogix (Web Client) portal by default, but it must be configured for it to show on entity Detail views in the Web Client.

Before you configure the ProcessTasksTasklet, create goal driven workflows bound to entities, and register them with the Process Host run time in the ProcessLib.xml file.

 

To configure ProcessTasksTasklet

  1. Open the SLXClient portal for editing. See "Editing a Portal" in the Application Architect online Help.

  2. Click the Tasklets tab, and select ProcessTasksTasklet.

  3. In the Display Mode drop-down list, select Detail.

  4. In the Entities drop-down list, select all entities for which you want to show the tasklet.

The entities you select should correspond to the entities to which you bound the workflows.

  1. Click away from the entity list to close it, and click Save.

Note: It is not necessary to rebuild the Web Client.

  1. Click the Dependencies tab, verify that the Process Host dependency is checked, and click Deploy All to redeploy both portals.

The Process Tasks pane has been added to the Detail views of the entities you selected in step 4. 

To use the Process Tasks pane

  1. Open the Detail view of an entity that has goal-driven workflows that are registered.

    The Process Tasks pane appears with hyperlinked names of the workflows that have not been started (instantiated) for the current entity. If the message "Process Host Not Found" appears, refer to the Process Orchestration FAQ help topic for more information.

  2. Start a workflow by doing one of the following:

  3. Click a workflow hyperlink in the Process Tasks pane.

  4. Cause a database change event that was defined to start a workflow. For example, database eventing might specify that a leads workflow starts when you create a new Lead.

Starting a workflow creates a workflow instance that is bound to the current entity instance. The workflow items - steps (tasks), stages, and scopes - appear in a tree view. The tree hierarchy reflects the relationships among the items defined in the goal-driven workflow.

  1. Mark a step as complete or cancel it by clicking one of the icons next to it.

    Once all of the steps in a stage are completed or cancelled, the stage is complete. You can complete a step outside the active scope. After all stages and scopes are complete, the workflow instance is deleted and the tree view is replaced by the workflow name.

This tasklet is a good example of how to provide a user interface to workflows in the Saleslogix Web Client. To view the tasklet code, open ProcessTasksTasklet  in Application Architect under Portal Manager > Saleslogix (Web Client) portal > SupportFiles > SmartParts > TaskPane > ProcessTasks.
