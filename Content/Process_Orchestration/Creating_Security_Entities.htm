Creating Security Entities

Create generated entities you can use when you create policy activities.

To create security entities

  1. Right-click on your Visual Studio project, click Add, and then click New Item.

  2. Create a new file named SecurityEntities.xml.

  3. Paste the following into the new XML file:

<?xml version="1.0" encoding="utf-8" ?>  

<EntityAccessGeneratorDefinition TargetNamespace="SecurityEntities">  

     <ConnectionString>[Your SLX Connection String Here]</ConnectionString>  

</EntityAccessGeneratorDefinition>  

  1. Replace the <ConnectionString> text with your Saleslogix connection string.

You can get your connection string from connection.config, which is located in the Deploy Path for a portal.

    

  1. In the Solution Explorer, select SecurityEntities.xml.

  2. In the Properties window, type SLXOwnersGenerator in the Custom Tool property.

  3. In the Application Architect Project Explorer tree view, check underneath the XML file for SecurityEntities.cs or SecurityEntities.vb. If neither exists, check Error List for possible reasons for the problem.
