Installing Saleslogix VS Workflow Items

The Saleslogix Visual Studio Workflow Extensions is a set of tools for creating Saleslogix-specific Windows Workflows.

To install       

  1. Ensure that Visual Studio 2008 is installed.

  2. On the Saleslogix DVD, browse to the Installs > Saleslogix VS Workflow Extensions folder, and double-click setup.exe.

  3. Click Next.

  4. On the License Agreement screen, read the information, accept the agreement, and then click Next.

  5. Select a setup type of Complete, and click Install.

  6. Click Finish.

  7. Consolidate assemblies so you can easily reference them in your Visual Studio project.

    To consolidate

    1. On the Application Architect Tools menu, click Build Settings.
      The Edit Build Settings dialog box opens.

    2. If necessary, in the Assemblies Path box, type or browse to a folder name where you want to copy assemblies, for example, C:\Program Files\SalesLogix\ReferenceAssemblies.

    3. Click OK.

    4. On the Application Architect Tools menu, click Copy Assemblies.
      Assemblies required by Process Orchestration are copied into the folder you specified.

      As with any other assemblies you reference in your Visual Studio workflow projects, reference the copy of Sage.Entity.Interfaces.dll in this folder because Application Architect automatically updates it each time you build interfaces or build the platform.

  8. Create Toolbox Items.

    To create

    1. Open the VS Toolbox.

    2. Right-click in the toolbox pane,  select Add Tab, and create a Saleslogix tab.

    3. Right-click the Saleslogix tab, and select Choose Items.
      The Choose Toolbox Items dialog box opens.

    4. Click the Activities tab, click Browse, and browse to the consolidated assemblies folder.

    5. Select Sage.SalesLogix.Process.Activities.dll, and click Open.
      This assembly contains the Saleslogix custom activities for sequential workflows.

    6. Make sure the activities in the Sage.SalesLogix.Process.Activities namespace are checked, and click OK.

    7. Right-click the Saleslogix tab, select Choose Items.

    8. Click the Activities tab, click Browse, and browse to the consolidated assemblies folder, for example, C:\Program Files\SalesLogix\ReferenceAssemblies.

    9. Select Sage.Platform.Process.dll, and click Open.
      This assembly contains the Saleslogix custom activities for goal-driven workflows.

    10. Make sure the activities in the Sage.Platform.Process.Activities namespace are checked, and click OK.

    Note: The tools display on the Saleslogix tab only after you open or create a project.
