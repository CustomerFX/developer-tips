Starting Workflows Through Database Eventing

Starting a workflow is equivalent to instantiating a workflow for a particular entity instance. You can configure a workflow to start in response to a database event such as the addition, change, or deletion of an entity. You can also set pre-conditions for the Process Orchestration run time to verify before running a process.

To set up

  1. Open the Process Host portal in Application Architect. See "Editing a Portal" in the Application Architect online Help.

  2. Double-click on ProcessStart.xml under SupportFiles.

  3. Create an <event> entry according to the examples documented in the file. Use the names of the workflow and the entity property (or properties) for which a change should start the workflow.

  4. Optionally include pre-conditions in the file.

  5. Save your changes and deploy them.
