Customizing a Saleslogix Web Help Topic

If you have customized views in the Saleslogix (Web Client) portal or Saleslogix Customer portal, you may want to customize the content of the associated help topic.

To customize a Saleslogix Web help topic

  1. In the Application Architect, open the portal for editing.
  2. In the Project Explorer tree view, expand SupportFiles\help\Content
  3. Open the subfolder that contains the help topic you want to edit, and then double click the help file.
    Help topics are organized in subfolders by subject. For example, accountdetailview.htm is in the Accounts folder.
  4. Customize help topic text, including adding or removing links or images.
    If you are upgrading customized help from a previous version, browse to the location where you saved your customized Web Help files before upgrading and recreate your customizations.
  5. Format the help topic text using the SalesLogix.css.
  6. Save your changes.
  7. Determine how your help topic can be found within the Help system. You can do any of the following:
    • Link the topic to a help button
    • Add the topic to the table of contents
    • Add the topic to the index
    • Edit other topics to link to this topic.
  8. Repeat steps 1-7 for each topic to be customized.

Your changes will not be visible until you deploy the changes to your Web site. If you link the topic to a quick form, the help button will not appear unless you do a full build and deploy the changes to your Web site. See the Application Architect online help topics "Rebuilding the Web Platform" and "Deploying the Saleslogix Web Client Portal".
