Working with the Alias.xml

In order to link a help topic to a help button, you must create an alias for the topic.

Files in a help directory are specific to the portal where they are stored. Changes must be made to each portal separately.

To open the alias.xml file

  1. In the Application Architect, open the portal for editing.

  2. In the Project Explorer tree view, expand SupportFiles\help\Data.

    If you are working with a child file, you must update the child file’s alias file located in: SupportFiles\help\Subsystems\<childfilename>\Data
  3. Double-click Alias.xml.

To add an alias

  1. Scroll to the bottom of the page and add the following before the </CatapultAliasFile> tag:
    <Map Name="alias" Link="folderlocation/topicname.htm" ResolvedId="75215334" />
  2. Replace alias with the alias for your topic. If you use the SmartPart name as the alias, then the help button will be linked without having to update the SmartPart Help Topic property.
  3. Replace topicname with the name of the help file you are adding.
  4. Replace folderlocation with the folder name (under Content) where you saved your custom help topic.
  5. Change the ResolvedId to a unique number.
  6. Save your changes.

Your changes will not be visible until you deploy the changes to your Web site.
