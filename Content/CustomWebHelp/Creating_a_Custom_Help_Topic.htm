Creating a Custom Help Topic

The easiest way to create a new help topic is to copy and then modify the samplehelp.htm topic. This will ensure that the necessary formatting and required information are contained in the new file.

To create

  1. Browse to the appropriate portal location, and open the help\Content\General folder.
  2. Copy the samplehelp.htm topic, keeping the information above the </head> tags. You may delete any content within the <body></body> tags, but do not delete the <body></body> tags.
  3. Change the information between the <title></title> tags to the title of the new help topic. This information is required.
  4. Add the custom help topic text, including any links or images.
    If you are upgrading customized help from a previous version, browse to the location where you saved your customized Web Help files and recreate your customizations.
  5. Format the help topic text using the SalesLogix.css.
  6. Save the new help file.
    • If you are creating a child file topic, save the file to the appropriate folder within the child file folder structure. For example, help\Subsystems\<childfilename>\content\<folder>.
    • If this is a new custom help topic, give the topic a unique and descriptive name. It is helpful to establish a naming convention when creating custom help files.
    • If this is an existing custom help topic, name the topic the same name used in the previous version, but change the file extension to .htm.  The file name is used as part of the link to a help button.

    7. When creating custom help for list views, you must use the following naming convention, where entity is the name of the entity associated with the custom list view: entitylistview.html Example: accountlistview.html
  7. Add the help file to the project.
  8. Create an alias if you want to link it to a help button.
  9. Determine how your help topic can be found within the Help system. You can do any of the following:
    • Link the topic to a help button
    • Add the topic to the table of contents
    • Add the topic to the index
    • Edit other topics to link to this topic
  10. Repeat steps 1-9 for each topic to be customized.

Your changes will not be visible until you deploy the changes to your Web site. If you link the topic to a quick form, the help button will not appear unless you do a full build and deploy the changes to your Web site. See the Application Architect online help topics "Rebuilding the Web Platform" and "Deploying the Saleslogix Web Client Portal".
