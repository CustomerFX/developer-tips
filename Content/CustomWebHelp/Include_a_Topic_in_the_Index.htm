Including a Topic in the Index

After creating a custom topic, you can create an index entry for the topic so that it appears in the index.

Files in a help directory are specific to the portal where they are stored. Changes must be made in each portal separately.

To include a topic in the index

  1. In the Application Architect, open the portal for editing.
  2. In the Project Explorer tree view, expand SupportFiles\help\Data.
  3. Double-click index.xml.
  4. Add the following text below the </IndexEntry> tag of the index entry that should appear above the new index entry. Saleslogix recommends placing index entries in alphabetical order.

     <IndexEntry Term="Test">

                <Entries />

                <Links>

                  <IndexLink Title="Test" Link="/Content/Test/test.htm#kanchor10000" UsedConditions="" />

                </Links>

              </IndexEntry>

  5. If you have previously edited the Index, note the last kanchor index number that was used.

    Index entries must be unique. Custom index entries should begin in the 10,000 range so that there will not be any conflicts with standard index entries.

  6. Replace the IndexEntry Term in quotes with the term you want to appear in the Index.
  7. Replace the Index Link Title in quotes with the title of the topic you are linking to.
  8. Replace the Content/Test.htm# with the new folder and file name.
  9. Replace the number after kanchor with the next available index number.
  10. Open the help file you are adding to the index and do the following:
    1. Add the following code in front of the text you want to index:
      <a name="kanchor10000">
    2. Replace 10000 with the number you used in step 8.
    3. After the index text, type </a>
    4. Save your changes to the help file.
  11. Repeat steps 2-10 for each index entry that you want to add.
  12. Count the number of index entries you added.
  13. Increase the number after <CatapultTargetIndex Count= (located near the top of the index.xml file) by the number of index entries you just added.
  14. Save your changes.

Your changes will not be visible until you deploy the changes to your Web site.
