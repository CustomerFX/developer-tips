Comparing Nulls in SData Queries

SData queries follow correct ANSI Null behavior. Because Null is undefined, it cannot be compared with non-null values. For example, the following where clause will not return records with DoNotSolicit = NULL:
 

where=DoNotSolicit ne "False"   

If you want to match both false and null value, use a composite expression, for example:
 

   where=not DoNotSolicit OR DoNotSolicit eq null
