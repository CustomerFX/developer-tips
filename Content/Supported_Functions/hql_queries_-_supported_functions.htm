Supported Functions for HQL Queries

Saleslogix supports a variety of functions for your use in all HQL queries. Some typical situations in which Saleslogixdevelopers build NHibernate queries are for HQL Data Source controls, for Mashup Query processors, and in code snippets. See the Application Architect online Help topics "Hql Data Source Control", "HQL Processor Type", and "Using Quick Form Action Items".

Common NHibernate functions

  • current_timestamp

  • year, month, day, hour, minute, second

  • concat

  • str

Functions supported by MSSQL and Oracle

  • sin, cos, tan, asin, acos, atan

  • abs, round, floor, sign

  • exp, log, sqrt

  • lower, upper, ltrim, rtrim

  • cast, coalesce, nullif

  • soundex

Functions mapped to the SaleslogixOLEDB provider scalar functions

In addition, Saleslogix has exposed the following functions to NHibernate through the Saleslogix NHibernate dialect:

  • slxgetdate (alias getdate) -  Returns GMT or UTC date and time.

  • slxdateadd (alias dateadd) - Returns a new datetime value based on adding an interval to the specified date.

  • slxdatediff (alias datediff) - Returns the number of time units in the interval between STARTDATE and ENDDATE.

  • slxdatename (alias datename) -  Returns a character string representing the specified datepart of the specified date.

  • slxdatepart (alias datepart) - Returns an integer that represents the specified part of the date.
