Work Items

A WorkItem represents a unit of work in an application. It is a container for user interface elements and views, state, and services. Every CAB application contains a Root WorkItem that serves as the top level container for global services and the other application Work Items. The definition of the WorkItem class resides in the Sage.Platform.Application assembly.

Accessing the WorkItem type through the ServiceDependency attribute causes your property to be associated with the WorkItem closest in the chain to the module in which your code resides. For example, services mapped to a portal are associated with the Root WorkItem. Smart Parts and modules mapped to a portal page are associated with the page or workspace WorkItem.
