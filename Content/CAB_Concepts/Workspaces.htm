Workspaces

Workspaces are user interface containers for Smart Parts. The Saleslogix portal page template contains several predefined workspaces: MainContent, Tab, Dialog, ToolBar, and NavigationBar. The separation of Smart Parts from their WorkSpace containers allows the appearance of an application to be changed through configuration with the Application Architect. When you map a Smart Part to a page, you choose the WorkSpace with which the Smart Part will be associated and thus where on the page the Smart Part will appear.

The Workspaces provided with the Saleslogix Web Client have their own WorkItems. When a portal page initializes the Smart Parts that are mapped to that page, the Smart Parts are added to CAB using the Workspace Show() method, which associates the Smart Parts with the Workspace WorkItem. This ensures separate entity contexts between Smart Parts on the same page.
