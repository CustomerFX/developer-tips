Composite UI Application Block (CAB)

The Saleslogix Web Client architecture uses Microsoft CAB objects. For example, it uses ObjectBuilder to implement the Builder and Dependency Injection patterns. Classes based on other CAB components are used to implement the other patterns in a platform-independent manner.

The following are some of the CAB objects most commonly used in Saleslogix Web applications:

  • ObjectBuilder

  • Modules - See "Understanding Modules" in the Application Architect online Help

  • WorkItems
  • Workspaces
  • Smart Parts - See "Understanding Smart Parts" in the Application Architect online Help
  • Services - See "What are Services" in the Application Architect online Help
