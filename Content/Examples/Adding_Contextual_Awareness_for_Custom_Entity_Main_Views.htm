Adding Contextual Awareness for Custom Entity Main Views

When users navigate from one main view to another, the current group list must be re-built based on the page the user navigated from. The rules about this contextual awareness are specified by entity in the ContextualListConfiguration.xml file for the portal. To add contextual awareness for navigating to and from custom entity main views, add them to the ContextualListConfiguration.xml file.

 

To add

  1. Open the support file, ContextualListConfiguration.xml, for the portal. For example:

     \Model\Portal\SLXClient\SupportFiles\ContextualListConfiguration.xml

  2. Add a <TargetConfigInfo> section to specify the context info for navigating to the new entity main view from other pages. For example, if you added a MYCUSTOMENTITY tab on the ACCOUNT main view, the user can navigate from ACCOUNT to MYCUSTOMENTITY. Create a new <TargetConfigInfo> section for MYCUSTOMENTITY:

        <TargetConfigInfo>

          <TargetPage>MYCUSTOMENTITY</TargetPage>

          <ReferingPages>

            <ContextConditionInfo>

              <ReferringPage>ACCOUNT</ReferringPage>

              <DataPath>MYCUSTOMENTITY:ACCOUNTID</DataPath>

              <ConditionValue>ReferringId</ConditionValue>

            </ContextConditionInfo>

            <ContextConditionInfo>

              <ReferringPage>Default</ReferringPage>

              <DataPath>MYCUSTOMENTITY:MYCUSTOMENTITYID</DataPath>

              <ConditionValue>CurrentId</ConditionValue>

            </ContextConditionInfo>

          </ReferingPages>

        </TargetConfigInfo>

  3. For each page to which you can navigate from the new main view, add a <ContextConditionInfo> section to the appropriate <TargetConfigInfo> section. For example, if there is a CONTACT tab on the MYCUSTOMENTITY main view, the user can navigate from MYCUSTOMENTITY to CONTACT. Add the following to the <TargetConfigInfo> section that contains <TargetPage>CONTACT</TargetPage>:

            <ContextConditionInfo>

              <ReferringPage>MYCUSTOMENTITY</ReferringPage>

              <DataPath>CONTACT:MYCUSTOMENTITYID</DataPath>

              <ConditionValue>ReferringId</ConditionValue>

            </ContextConditionInfo>

  4. If you edited the file in the model, redeploy the portal. See "Redeploying Portals" in the Application Architect online Help. If you edited the file directly in the portal, restart the Web server and remember to port your changes back to the model.
