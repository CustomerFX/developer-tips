ContextualListConfiguration.xml File

When users navigate from one main view to another, the current group list must be re-built based on the page the user navigated from. The rules about this contextual awareness are specified by entity in the ContextualListConfiguration.xml file for the portal. It has the information needed by the group classes to build up the condition for the lookup results group.  For each entity's main view, there is a <TargetConfigInfo> section that defines the conditions based on the <ReferingPages>.

Review ContextualListConfiguration.xml at Model\Portal\SLXClient\SupportFiles\ as an example. Its <TargetConfigInfo> section for the SLXClient portal Ticket main view is reproduced below with comments. This <TargetConfigInfo> section specifies the group list rules when navigating to the Ticket main view.

 

Line in XML file

Comment

<TargetPages>

  

    <TargetConfigInfo>

  

      <TargetPage>TICKET</TargetPage>

The detail view the user is navigating to – TICKET.

      <ReferingPages>

  

       <ContextConditionInfo>

  

          <ReferringPage>ACCOUNT</ReferringPage>

The detail view the user is navigating from – ACCOUNT.

         <DataPath>TICKET:ACCOUNTID</DataPath>

Path to column of the current table to check against ConditionValue.

          <ConditionValue>ReferringId</ConditionValue>

The condition the column/field must meet for a record to be included in the group. The options are:

• ReferringId – include all records related to the referring entity.

• CurrentId - include just the current entity, which creates a group of one record.

        </ContextConditionInfo>

  

        <ContextConditionInfo>

Context for navigating from CONTACT to TICKET starts here.

          <ReferringPage>CONTACT</ReferringPage>

  

          <DataPath>TICKET:CONTACTID</DataPath>

  

          <ConditionValue>ReferringId</ConditionValue>

  

        </ContextConditionInfo>

  

        <ContextConditionInfo>

Context for navigating from anywhere else to TICKET starts here.

          <ReferringPage>Default</ReferringPage>

  

          <DataPath>TICKET:TICKETID</DataPath>

  

          <ConditionValue>CurrentId</ConditionValue>

(Note use of CurrentId as condition.)

        </ContextConditionInfo>

  

      </ReferingPages>

  

    </TargetConfigInfo>
