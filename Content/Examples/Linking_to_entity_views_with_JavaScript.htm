Linking to Entity Views with JavaScript

Most entity details are rendered in a main view. However, a few entities such as Activities and ActivityHistory load in a dialog box. Thus, when you build a URL to redirect to entity detail pages, you must change the URL pattern depending on the entity type.

A more convenient approach is to use the client-side method, Link.entityDetail, which can accept any entity type and an ID and direct the user to it in the appropriate way. For example, the following call will redirect to the Account detail page for the specified entity:

 

javascript:Link.entityDetail('ACCOUNT', 'AGHEA0002669');

 

While this call will open the Activity dialog for the specified entity:

  

javascript:Link.entityDetail('ACTIVITY', 'VDEMOA00006N');

 

Using Link.entityDetail also eliminates the task of building up the URL string. It is particularly useful when generating a list of links to views of mixed entity types, for example the results from Speedsearch.
