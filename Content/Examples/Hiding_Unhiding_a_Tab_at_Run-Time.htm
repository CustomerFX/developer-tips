Hiding/Unhiding a Tab at Run Time

You can hide and unhide a tab using code in a page module. The portal page loads modules as one of the last parts of the page creation process.  Using a custom module to set the hidden flag on a tab and placing it after the other modules will ensure that the change is the last one executed.

  1. Create a custom module to hide and show a tab. Tell me more...

    The following sample code hides the Contacts tab on the Account detail page when the account type is "Customer".

    //Reference these assemblies in your project:

    //     Microsoft.Unity.dll.

    //     Sage.Entity.Interfaces.dll

    //     Sage.Platform.Application.dll

    //     Sage.Platform.Application.UI.Web.dll

    //     Sage.Platform.WebPortal.dllProperties

  2. Add you assembly. to the portal support files. For example, under Saleslogix (Web Client) portal\SupportFiles\Bin.
    See "AddingExisting Support Files" in the Application Architect online Help.
  3. Add the module to the portal page by doing the following:
    1. Open the page for editing. For example, the Account Detail page of the Saleslogix (Web Client) portal.
    2. Click the Modules tab, and add the module to the end of the list. See "Adding Modules" in the Application Architect online Help.
    3. Set Run in Mode = Detail.
    4. Click Save All.
  4. Build and deploy. See the Application Architect online help topics "Building the Web Platform" and "Deploying the Saleslogix Web Client Portal".
