Customizing the What's New Module

You can customize the tabs on the What's New page by configuring an XML file. When the What's New module is executed within the page, it looks for the existence of WhatsNew_Config.xml in the App_Data folder of the portal. If it exists, the rendering logic changes and uses the details contained within to determine the smart parts that get displayed. A sample configuration file, WhatsNew_Config_Example.xml, is provided. The easiest way to configure the file is to deploy your Saleslogix (Web Client) portal, copy or rename the resultant file to WhatsNew_Config.xml, and edit it in Visual Studio.  When you have completed your changes in Visual Studio, copy the updated file back into the SupportFiles\App_Data folder in your project.

To configure

  1. Browse to the App_Data folder of the SlxClient portal on the Web server, for example, C:\Inetpub\wwwroot\SlxClient\App_Data.
  2. Open WhatsNew_Config_Example.xml with the editor of your choice and save as WhatsNew_Config.xml in the same folder.
  3. Ensure that for each tab you want on the What's New form, there is a section in the XML file that looks similar to the following. Add or delete sections as necessary:

      <item targets="remote" window="main" enabled="true">

        <virtualPath>~/SmartParts/WhatsNew/RemoteMainWhatsNew.ascx</virtualPath>

        <partId>RemoteMainWhatsNew</partId>

        <title>Accounts</title>

        <description>Accounts</description>

      </item>

  4. For each tab you want to customize, do the following:
    1. Change any of the following item node attributes:

      targets

      Determines if the smart part is to be used in the local or the remote system. Possible values are 'server' and 'remote'.

      window

      Indicates the workspace where the smart part should be rendered. 'main' specifies the header while any other value will target the tab workspace.

      enabled

      Indicates if the smart part should be displayed. True or False.

    2. Change any of the following item node children:

      <virtualPath>

      Points to the location of the smart part relative to the web site. Should begin with ~/SmartParts/

      <partId>

      Value of Smart Part Id in Application Architect. Must be unique within this XML file.

      <title>

      Title to display for the smart part.

      <description>

      Description of the smart part.

  5. Save the XML file.
  6. Add or replace the XML file in the model so the changes will be retained in future deployments.
  7. Deploy to other targets such as your remote Web server as appropriate.
