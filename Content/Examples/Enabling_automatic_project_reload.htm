Enabling automatic project reload

Portals get the following information directly from the Application Architect project model, not from files in the deployed web site:

  • Form metadata used by the Web Client Form Designer
  • Mashup metadata
  • Filter metadata

A portal can be set up to automatically reload the project if it is changed, for example in Application Architect. Automatic project reload is enabled by default for the SLXClient portal.

For VFS-based projects, th eSaleslogix Messaging Service must be running for notification of project changes to be received.

To enable automatic project reload for a portal

  1. In the Application Architect, open the portal. See "Opening the Portal Editor" in the Application Architect online Help .
  2. Click the Modules tab, and select Sage.SalesLogix.Web.ProjectsModule.
  3. Click the Configure button, set EnableAutoReload=True, and click OK.
  4. Deploy the portal.
