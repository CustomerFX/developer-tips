Improving Performance of Custom JavaScript

The Saleslogix common scripts are minified and merged during the build process into Sage-common.js (or sage-common-debug.js) in order to reduce the number of http requests to the server.

You can add your custom JavaScripts to this project to realize the same benefit.

Add to the project only scripts that you would normally reference in the master pages. If you are using dynamic script addition through ScriptManager, do not include the script in the minify and consolidate process.

During the build process, the jsb project file is used to generate sage-common.js and its debug version, sage-common-debug.js.

Saleslogix recommends that if you must change the Saleslogix script files that are merged, do not edit the Saleslogix files, but instead, override in your custom script.

Sage-common.js and sage-common-debug.js are generated files. They contain anything that is in the files listed in SageSalesLogixCommon.jsb.  Jsb files are project files created by JSBuilder which is a free utility from Google.

To add

  1. Add your script to SageSalesLogixCommon.jsb in the model by doing one of the following:
    • Edit the XML file in Application Architect. How?
      1. Open the Portal Explorer and expand the node for the portal.
      2. Double-click SageSalesLogixCommon.jsb in the SupportFiles/jscript folder.
      3. Add a <file> and <include> nodes for the script you want to add.
      4. Save and close the file.
    • If the project is a local project workspace, you have the option to use the JS Builder utility. Tell me more...
      • The JS Builder utility is available for download at http://code.google.com/p/js-builder/.

      • SageSalesLogixCommon.jsb is located in the project workspace folder at Model\Portal\SlxClient\SupportFiles\jscript.

  2. Build and deploy the portal.

    If you choose to edit the file in the deployed website files instead of in the project workspace model, remember to copy the altered file back into the model or it will be overwritten in subsequent portal deployments.
