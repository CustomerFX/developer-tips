Troubleshooting Data Binding Errors

In Application Architect, the data binding editor attempts to validate data binding elements as you set them. However, since there are some things it cannot evaluate until run time, it may generate breaking data binding elements. To see data binding errors that happen at run time, turn on the ShowBindingErrorsAtRuntime flag in the appSettings.config file for the portal. This visibility can be particularly useful when debugging custom smart parts.

To turn on

  • Edit the appSettings.config file with the following changes:
    1. Locate the following line in the <appsettings> node:
      <add key="ShowBindingErrorsAtRuntime" value="false"/>
    2. Replace "false" with "true".
