Setting the Active Tab and Captions on a MultiTab Control

On a MultiTab control, the user can click a tab to set the visible page, or you can set the visible page through code. The following examples show how to set some Multitab properties through code. The first example explains how to set the active tab, and the second example shows how the captions are dynamically updated to the opportunity description and the account name.

To set the active tab

  1. Insert a MultiTab control onto an Opportunity bound quick form. See " MultiTab Control" in the Application Architect online Help.
  2. Add a second tabbed page to the control and add multiple controls to each page.
  3. Leave the captions undefined so the pages will not have visible tabs for users to click.
  4. Use the FormActionDefinition Collection Editor to add a Code Snippet Action Item with the following property values:
    • On Entity Changing = True
    • On Repaint Event = False
  5. Create the code snippet and open it for editing. See "Editing Action Item Code Snippets" in the Application Architect online Help..
  6. Insert the following code into the snippet.

     

         Sage.Entity.Interfaces.IOpportunity opp = form.CurrentEntity as Sage.Entity.Interfaces.IOpportunity;

              if (opp != null) {

                 if (form.QFMultiTab.Tabs.Count > 1) {

                               form.QFMultiTab.ActivePageIndex =(opp.Description.Contains("Abb")) ? 1 : 0;

                                     

                                                     }

                                }

  7. Close the snippet, and save the form.
  8. Add the quick form to the tab workspace on the Opportunity detail page. See "Mapping Smart Parts to Workspaces" in the Application Architect online Help.
  9. Build and deploy. See the Application Architect online help topics "Building the Web Platform" and "Deploying the Saleslogix Web Client Portal".
  10. In the Saleslogix (Web Client) portal, browse to the Opportunity detail page, and then open the tab with the new quick form. If the Opportunity Description contains “Abb”, then the second page of the MultiTab control becomes the active page. If it does not, the first page of the control becomes the active page.

To set captions

  1. Using the MultiTab control defined in the previous example, specify a value for the ActivePageCaption property of each page of the control. Specifying captions for the pages causes their tabs to show. See "ActivePageCaption" in the Application Architect online Help.
  2. Add the following lines to the code snippet to set the captions dynamically:              

              form.QFMultiTab.Tabs[0].Caption = opp.Description;

              form.QFMultiTab.Tabs[1].Caption = opp.Account.AccountName;

    Type the Tabs[x].Caption property if it does not appear in the code complete.

  3. . Build and deploy. See the Application Architect online help topics "Building the Web Platform" and "Deploying the Saleslogix Web Client Portal"
