Using the Global CSS Image Sprite

Saleslogix uses an image sprite to reduce the number of server requests for icons. User interface elements created with Application Architect use the icons by automatically referencing the appropriate sections of the sprite.  Customizations can also use the global image sprite to reduce server requests.

To use

  1. Use the Image Editor to add an image of size 16X16, 24X24, or 32X32 to the project or to identify an existing one you want to use.
  2. Build the Web platform. See the Application Architect online help topic "Building the Web Platform".
    All images of size 16X16, 24X24, or 32X32 are generated into a single image. Also generated is a style sheet that contains a class for accessing each icon as well as the classes, Global_Images, icon32x32, icon24x24, and icon16x16.
  3. Deploy the portal. See the Application Architect online help topic "Deploying the Saleslogix Web Client Portal".
    The sprite is copied to <portal>\images\icons\Global_Images.png and the style sheet to <portal>\css\Global_Images.css.
  4. In your code, access the icon using three classes named Global_Images, icon<sizexsize>, and icon_<yourimagename>.
    For example, this is the HTML markup for the welcome icon in the navigation bar:
    <div class="Global_Images icon32x32 icon_welcome_32x32"></div>

    As a JavaScript example, look at the cancel button in OpportunityProducts.js:
    imageClass: 'icon_reset_16x16'

    The imageClass property was added to Sage/UI/ImageButton, Sage/UI/MenuItem, and Sage/UI/MenuBarItem for the client-side framework. Just set its value to 'icon_<yourimagename>'; it adds the Global_Images and icon<sizexsize> class names for you.
  5. View the user interface.
    The first reference to any icon requests the sprite from the server. Subsequent icon references use the same sprite so no additional server request is necessary.
