Exposing Tabs on Page Load

When a page loads, the tabs are exposed based on the active tab and the remembered tab state. To make a particular tab visible, use one of the following methods according to how the page will be accessed:

To override for a page accessed through a link

  • On the Querystring, add &activetab=mySmartPartID.

This activates the tab named "mySmartPartID" so that it is open wherever it is mapped (middle pane, main tab area, or MoreTabs tab). If a user accesses the page in a way other than through the link, the remembered tab state is displayed. This method works on any page with a tab workspace.

Example  Link from the "Reminders" view to the Activity List view with the "Confirmations" tab open.

To override for initial page load

  • In the template properties of the portal page, set the OverrideActiveTab property to mySmartPartID.

This causes the tab named "mySmartPartID" to be open every time the page loads, regardless of how the user accesses the page. This method works only if the master page exposed the property.

Example Load the Schedule Activity page with the Members tab open.
