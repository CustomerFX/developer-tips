Setting Potential Match Indexing Options

The Potential Match Service is used during lead import to find potential matches for the lead being imported and to prevent duplicates in the system. You can set options that specify what situations require a Potential Match Index update.

Search for Potential Matches

When you search for duplicates while adding a new contact, account, or lead or converting a lead, incremental indexing is not performed before the search unless you enable the TurnOnIndexing option in the PotentialMatchConfigurations.xml file. Tell me more...

To enable

  1. Open the VFS Explorer.
  2. Expand Configuration > Global.
  3. Double-click on PotentialMatchConfigurations.xml.
    The xml file is opened for edit.
  4. Find the <AdvancedOptions> section.
  5. Find or add the <TurnOnIndexing> entry.
  6. Set the value to true. For example:
    <TurnOnIndexing>true</TurnOnIndexing>
  7. Save the file.

This configuration is cached. You must restart the web site before the change takes effect.

Import Configurations

By default, when you run a lead import and search for duplicates, incremental indexing before the import is enabled and updating the index during the import is disabled. Indexing will be performed during the import if the user selects the "Check for duplicates contained within the import source file" option in the Lead Import dialog box or you enable the IndexAfterInsert option in the ImportConfigurations.xml file. Tell me more...

To enable

  1. Open the VFS Explorer. See "Opening the VFS Explorer" in the Application Architect online Help.
  2. Expand Configuration > Global.
  3. Double-click on ImportConfigurations.xml.
    The xml file is opened for edit.
  4. Find the <ImportAdvancedOptions> section.
  5. Find or add the <IndexAfterInsert> entry.
  6. Set the value to true. For example:
    <IndexAfterInsert>true</IndexAfterInsert>
  7. Save the file.

This configuration is cached. You must restart the web site before the change takes effect.
