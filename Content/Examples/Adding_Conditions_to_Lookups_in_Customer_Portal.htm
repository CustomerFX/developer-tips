Adding Conditions to Lookups in Customer Portal

You can narrow the scope of lookups on an entity in the Saleslogix Customer portal by adding a condition in the appSettings.config file.

To add

  1. Edit the appSettings.config file for the Saleslogix Customer portal.
  2. Locate the <appSettings> section in the script.
  3. Insert a new line after <add key="LookupControlMaxResults" value="500"/>
    For example, use the following statement to limit the search results to tickets received after 3/1/2007.
    <appSettings>
    <add key="LookupControlMaxResults" value="500"/>
    <add key="CustomerPortalLookupConditions_TICKET" value="RECEIVEDDATE &gt; '03/01/2007 1:00:00 AM'"/>
  4. Save the changes and close the file.

    You can use multiple clauses in the same parameter. XML encoding must be used for filtering. The expression (<>) should be written as (&lt;&gt;). For example:
    <add key="CustomerPortalLookupConditions_TICKET" value="status = 'Open' and PUBLICACCESSCODE &lt;&gt; 'k6UJ9A0000OW' "/>
