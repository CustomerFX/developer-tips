Applying Field Level Security to Entity Insertion

Field Level Security, by default, does not apply to insertion of new entities. Take for example a user whose Field Level Security specifies no access to the CreditRating property of the Account entity. The user can set the CreditRating property during the insert of a new Account. After the insert, the user cannot change or read the property.

You can override the default behavior at the application level so that Field Level Security applies to record inserts. Then the user in the example would still be able to insert the new Account, but any value provided for the CreditRating property would be ignored during the insert.

To override

  • Edit the appSettings.config file with the following changes:
    1. In the appsettings section, locate the value for the ApplyFieldLevelSecurityOnInsert key.
    2. Modify it. For example:
      <add key="ApplyFieldLevelSecurityOnInsert" value="true">
